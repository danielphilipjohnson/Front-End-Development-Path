Basic HTML

List of Basic HTML tags

  • <!DOCTYPE> Defines the document type
  • <html> Defines an HTML document
  • <head> Contains metadata/information for the document
  • <title> Defines a title for the document
  • <body> Defines the document's body
  • <h1> to <h6> Defines HTML headings
  • <p> Defines a paragraph
  • <br> Inserts a single line break
  • <hr> Defines a thematic change in the content
  • <!--...--> Defines a comment

Why we use <!DOCTYPE>

It tells the browser what document to expect the web browser sniffs the doctype and runs its HTML layout engine

How to use it

This is all you need to put in HTML5

<!DOCTYPE html>

Why we use <html>

Because <html> represents the root(top-level element) of an HTML document and it is the container for all the other HTML elements(descendants of this element) we are going to learn from here on

How to use it

<html lang="en">

It is important to include the attribute lang inside the html tag as it assist search engines and browsers. Also assists with reading technology being able to determine what language to read the page!

Why we use <head>

This element contains machine-readable information (metadata) about the document, like its title, scripts, and style sheets.

In english? This is the container where we put the title of the page and scripts to make the page interactive and to make the website look nice.

You will learn these things later (scripts and styling).

  • <title> (required in every HTML document)
  • <style> </style>
  • <base>
  • <link>
  • <meta>
  • <script> </script>
  • <noscript> </noscript>

How to use it

Adding a title to webpage

<head>
<title>This is my title</title>
</head>

Adding a style tag

<head>
<style>p {text-align: center}</style>
</head>

It is best to avoid littering your html file with css syntax. It is best to include in a stylesheet. There are some edgecases such as a CSS rule that is complex and takes a while to run so putting it in html might speed up the rendering.

Adding a base tag

<head>
<base href="https://www.example.com/" >
</head>

This specifies the base URL and/or target for all relative URLs in a document. In this example if you set your base html link to https://www.example.com/ when you set a link to another page coolcats it will go to https://www.example.com/coolcats

Adding a stylesheet

<head>
<link rel="stylesheet" href="style.css">
</head>
Adding meta tags

The meta tag content is never displayed on the page but instead it is read by the browser(on hints on how to display content or to reload the page) and search engines for keywords. There is various attribute tags that can be used, some are self explanatory for instance author and the description of your website. Please do not go to crazy with the description keep it to about 160 characters otherwise search engines might punish your website. Keywords will be words that describe your site you could consider this to be the hashtags you use when posting on instagram content="Tech, Blog, Techblog". Just like instagram do not go to crazy with keywords. Viewport is important for making your site look good on every device. It isnt magic and will not make make website reponsive for that you will need CSS. However, the viewport helps with how the webpage is layed out across devices initial-scale=1.0 refers to fill 100% of screen. It prevents scrolling to zoom on mobiles or tablets

Adding a meta tags to webpage
<head>
<meta charset="UTF-8">
<meta name="description" content="Free Web tutorials">
<meta name="keywords" content="HTML, CSS, JavaScript">
<meta name="author" content="John Doe">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>

Adding a script

<head>
<script src="index.js"></script>;
</head>

Adding a no script

Defines a section of HTML to be inserted if a script type on the page is unsupported or if scripting is currently turned off in the browser.

<head>
<noscript> Your browser does not support JavaScript! </noscript>
</head>

Why we use <body>

Defines the document's body. This is where we put our content. Element represents the content of an HTML document. There can be only one <body> element in a document.

How to use it

<body> <p> I am a paragraph</p>
</body>

Why we use <h1> to <h6>

We use these to create headings. Typically you will only use h1 once in a document. Think of this as your book title! It is best to use <h1> to h6 as section headings. Do not avoid skipping heading levels: always start from <h1>, next use <h2> and so on.

How to use it

<h1> Book title </h1> <h2> Chapter </h2> <h3> Sub topic </h3>

Why we use <p>

Defines a paragraph

Paragraphs are usually represented in visual media as blocks of text separated from adjacent blocks by blank lines and/or first-line indentation, but HTML paragraphs can be any structural grouping of related content, such as images or form fields.

Browsers automatically add a single blank line before and after each

element.

How to use it

<p> Paragraph text </p>

Why we use <br>

Inserts a single line break

It is useful for writing a poem or an address, where the division of lines is significant.

How to use it

<p> Paragraph
<br> text
</p>

Why we use <hr>

Defines a thematic change in the content

It is useful for writing a poem or an address, where the division of lines is significant.

How to use it

<p> This is the first paragraph of text.</p> <hr> <p> This is the second paragraph of text.</p>
